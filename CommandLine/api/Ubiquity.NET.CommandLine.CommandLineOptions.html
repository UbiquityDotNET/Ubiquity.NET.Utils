// new syntax enabled by C# 14
MyOptions.BuildRootCommand( settings, (o)=>1);

// Legacy syntax is still valid
CommandLineOptions.BuildRootCommand<MyOptions>( settings, (MyOptions o)=>1);

Remarks

information

If C# 14 and the `extension` everything feature is supported these will leverage it such that the class name is not needed. Otherwise, for older targets the type name is required. While default static methods on an interface is allowed by the language, use of them requires explicitly using the interface name and not the type. Additionally, since the interface is generic, this runs afoul of CA1000: Do not declare static members on generic types. Thus, this compromise is used to at least simplify the usage as much as possible. Direct use of these as static methods is valid syntax in both runtimes so is the safest approach for code that targets multiple runtimes.
